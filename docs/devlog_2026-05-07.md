# DevLog: 7 May 2026

## Overview
Today's session focused on resolving a race condition in our background processing, closing the loop on our visual evidence UI, and applying several UI polish updates for the Flutter mobile app.

## Key Accomplishments

### 1. AI Inference Pipeline Resiliency
- **Race Condition Fixed:** Addressed an issue where the Go background worker would frequently default to "Processing..." due to attempting to fetch the image from Cloudinary before CDN propagation completed.
- **Increased Tolerance:** Raised the worker context timeout from 30s to 60s and increased HTTP fetch retry attempts (to 5) and the delay (to 2 seconds).

### 2. Resolution Evidence UI (Flutter)
- **Model Update:** Added `afterImageUrl` to the `GrievanceReport` model, parsing it directly from the backend's JSONB metadata.
- **Side-by-Side View:** Updated `ReportDetailSheet` to dynamically render the "Before" and "After" photos side-by-side in a `Stack` when a task is marked `RESOLVED`.

### 3. "Resolved By" Accountability Tracking
- **Backend Context:** Updated the JWT claims and middleware to extract the worker's `university_id` and inject it into the request context.
- **Metadata Update:** The `UpdateStatus` handler now records `resolved_by_university_id` in the report's metadata when a task is successfully resolved.
- **Admin Visibility:** Admins now see a distinct, green "Resolved by: [ID]" tag below the "Reported by" section in the mobile app.

### 4. UI Polish & Overflow Fixes
- **Dashboard Header:** Fixed RenderFlex overflow issues occurring with long usernames by utilizing `maxLines: 1`, truncating text with ellipses, and shrinking the "Manage Users" button.
- **Report List Cards:** Applied `Flexible` and `Expanded` widgets to the report IDs and GPS coordinates in the report list to prevent them from crashing into status pills and timestamps on smaller screens.

## Pending Issues (For Next Session)
- **Hard Geofencing:** Implement strict campus boundaries using PostGIS polygons to block off-campus reports.
