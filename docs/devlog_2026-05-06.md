# DevLog: 6 May 2026

## Overview
Today's session focused on architectural hardening and workforce accountability. We moved from a server-side image proxy model to a **Client-Side Pre-signed URL** strategy, significantly reducing bandwidth load on the Go backend. We also finalized the "Proof of Work" anti-fraud mechanism for workers.

## Key Accomplishments

### 1. Robust Storage Architecture (Cloudinary)
- **Backend Signature API:** Implemented `/api/storage/signature` to generate secure SHA-1 hashes for direct frontend uploads.
- **Unified Media Service:** Updated Flutter to handle direct-to-Cloudinary uploads for both students (reporting) and workers (resolving).
- **Backend Refactor:** Converted `POST /api/reports` from `multipart/form-data` to a lightweight JSON API.

### 2. Workforce "Proof of Work" (Anti-Fraud)
- **GPS Validation:** Backend now uses PostGIS `ST_Distance` to enforce a 50m proximity rule for task resolution.
- **Frictionless UX:** Resolution now triggers an instant camera capture + silent background GPS geotagging in the mobile app.
- **Evidence Storage:** 'After' image URLs and resolution distance are now saved in the report's JSONB metadata.

### 3. Admin & UI Hardening
- **Global Visibility:** Fixed a bug where Admins were trapped in the 5km local geofence; they now use the global `/api/reports/all` view.
- **CSV Import UI:** Wired up the `file_picker` package to the Admin Dashboard for bulk user onboarding.
- **Overflow Fix:** Wrapped the dashboard greeting in an `Expanded` widget to prevent RenderFlex errors with long usernames.

## Pending Issues (For Next Session)

- **Resolution Evidence UI:** Update the `ReportDetailSheet` to display the "After" photo for resolved tasks (currently only showing the 'Before' image).
- **Hard Geofencing:** Implement strict campus boundaries to block off-campus reports (Postponed for testing convenience).
